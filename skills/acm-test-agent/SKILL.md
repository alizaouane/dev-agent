# acm-test-agent

Generates one failing test per acceptance criterion. Runs in a context strictly separated from the implement-agent so the same model session is not writing both the impl AND the tests it must pass.

## Inputs

- `<spec_path>` — path to the approved spec (raw markdown)
- `<criteria>` — pre-extracted `[{id, text, ...}]` from `lib/acm.ts`'s extractor
- `<test_framework>` — detected from `commands.test` (`vitest` / `jest` / `pytest` / `rspec` / ...)
- `<test_dir>` — output dir (`tests/acm/` by default)
- `<public_api_summary>` — typed summary of the consumer's public surface (function signatures, route shapes, schema definitions) — generated by indexing the consumer's exported declarations. **No implementation bodies.**

## Context isolation

The phase workflow MUST invoke this skill with `--allowedTools Read tests/ public-api/ Write tests/acm/ Bash(npm test:* tests/acm/)`. Reading any path outside that allowlist is a hard fail (read-scope manifest enforced at the Bash-tool layer in v1.1+; for v1, enforced via tool allowlist + drift-check on phase artifacts).

## Behavior

For each criterion:

1. Choose the right file boundary (one test file per criterion, named `<issue>-<id>-<slug>.<ext>`).
2. Emit a single failing assertion that *would* pass if the criterion were correctly implemented. Prefer one assertion per test — multiple assertions hide which fault the test caught.
3. Use only fixtures + helpers reachable from the public-API summary. No mocks of internal collaborators (Augment's anti-pattern: tests-with-impl-mocked pass without actually exercising the impl).
4. The assertion **must fail** when run against the current `main`. Verify by running `commands.test -- tests/acm/<file>` before declaring success.
5. Emit the per-test mapping `{criterion_id, test_file, test_name}` for the manifest builder.

## Output

A directory of `tests/acm/*` files plus a structured handoff to the acm orchestrator:

```json
{
  "criteria_bindings": [
    {
      "criterion_id": "AC-1",
      "test_file": "tests/acm/{{issue}}-1-health-returns-200.test.ts",
      "test_name": "AC-1: GET /health returns HTTP 200",
      "rationale": "<1-2 lines on which assertion catches failure>"
    },
    ...
  ],
  "discarded_tests": [
    { "criterion_id": "AC-3", "reason": "passed on first run — stub was vacuous" }
  ]
}
```

## Discipline

- Tests must be **deterministic**. Time-of-day, random seeds, network calls → wrap in fixtures or mark `skip` with a TODO.
- Tests that need cross-criterion fixtures → put fixtures in `tests/acm/_fixtures/` and document in `tests/acm/README.md`.
- **No implementation hints** in test bodies. The test asserts *behavior*; the implement-agent figures out *how*.

## Cost

Uses `models.acm_test_agent` (defaults to the same dated Sonnet snapshot as `models.acm`). Bundled inside the per-spec ACM cost cap.
